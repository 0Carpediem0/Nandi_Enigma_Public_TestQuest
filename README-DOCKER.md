# Запуск демо в Docker

## Что будет

- **Сборка** — 2–5 минут (ставятся только FastAPI, uvicorn, psycopg, без torch/transformers).
- **После `docker compose up --build -d`** поднимаются PostgreSQL и бэкенд с фронтом на порту 8000.

## Шаги

1. Убедиться, что есть файл `backend/.env` (хотя бы с переменными БД; для демо подойдёт и ваш текущий).
2. В корне проекта:
   ```bash
   docker compose up --build -d
   ```
3. Открыть в браузере:
   - http://localhost:8000 — веб-таблица
   - http://localhost:8000/operator.html — панель оператора

## Важно

- В контейнере **Qwen отключён** (переменная `QWEN_ENABLED=false`), чтобы не ставить тяжёлые библиотеки. Для демо жюри достаточно веб-интерфейса и БД; заглушки «В разработке» показываются там, где нужна почта/ИИ.
- Если нужен полный образ с ИИ (torch, transformers), можно собрать вручную с `backend/requirements.txt` и без `requirements-docker.txt` — тогда сборка займёт заметно больше времени.
